WayMark Privacy Policy

Last updated: March 2, 2026

1. Overview

WayMark helps users interact with data stored in Google Sheets and Google Drive. This policy explains what information is processed and how it is used.

2. Browser-First Architecture

Nearly all data processing happens in your web browser. The WayMark server serves the application files, brokers OAuth authentication, and provides a URL-fetch proxy for recipe imports (see Section 5). It does not receive, inspect, or store your Google Sheets content, Drive files, or any other user data. Your Google Sheets and Drive data is fetched directly from Google's APIs by JavaScript running in your browser. Template detection, data analysis, import processing, and rendering all happen client-side.

3. Information We Process

WayMark processes the following in your browser:

  • Google account basic profile information (name, email, profile image)
  • OAuth tokens required to access approved Google APIs
  • Google Drive and Google Sheets metadata and content needed for app features
  • Data you import from external files or URLs (processed entirely client-side)

WayMark does not sell personal information.

4. Google API Data Use

WayMark uses Google APIs only to provide core app functionality requested by the user. All Google API calls are made directly from your browser to Google's servers. WayMark's use and transfer of information received from Google APIs adheres to the Google API Services User Data Policy, including Limited Use requirements.

5. Data Import & Recipe Scraping

WayMark lets you import data from Google Sheets, Google Docs, and recipe URLs. For Google Sheets and Docs, your browser reads the source directly via Google APIs, analyzes its structure, and writes converted data to a new Google Sheet — all client-side.

When you import a recipe from a URL, the page HTML is fetched through WayMark's server (acting as a CORS proxy) and returned to your browser, where all parsing and data extraction happens client-side. The server does not store, log, or inspect the fetched content — it relays the page HTML and discards it immediately. No imported content is permanently stored on WayMark's servers.

6. Session Handling

OAuth is handled through Google. Access tokens are short-lived and held in browser memory. Refresh tokens may be stored in secure, HTTP-only cookies managed by the server. The server never uses these tokens to access your data — they are returned to your browser for client-side API calls.

7. Data Storage

  • Google Drive: Your files and WayMark user preferences are stored in your own Google Drive account.
  • Browser localStorage: UI preferences (sidebar state, tutorial progress, sort order) and usage context (recently opened sheets, search history, pinned folders, import history) are stored locally on your device and never transmitted to WayMark's servers. This data is also synced to a preferences file in your Google Drive if you are signed in.
  • WayMark server: Stores nothing — no database, no cache, no session store, no user data.

8. Data Sharing

WayMark does not share your data with third parties except: Google services you authorize through OAuth, infrastructure providers required to run the app, or where required by law. Because all processing is browser-based, your file contents are never accessible to WayMark infrastructure.

9. Security

WayMark applies reasonable safeguards such as HTTPS in production, secure cookie settings, and Content Security Policy headers. The browser-first architecture limits your data exposure to your own browser session and Google's APIs. No method of transmission or storage is 100% secure.

10. Your Choices

You can revoke access anytime in your Google Account permissions settings, clear browser localStorage to remove local preferences, and stop using the app at any time.
